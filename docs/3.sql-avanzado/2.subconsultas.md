Una **subconsulta** es una instrucción `SELECT` que se encuentra anidada dentro de otra instrucción SQL (conocida como instrucción contenedora o consulta externa). Se utilizan para filtrar datos, generar valores o construir conjuntos de datos temporales, funcionando en esencia como una consulta dentro de otra.

Sus características principales y ejemplos:

### Características y Tipos
*   **Sintaxis:** Siempre deben ir encerradas entre **paréntesis**.

*   **Ejecución:** Por lo general, la subconsulta se ejecuta antes que la consulta externa para que sus resultados puedan ser utilizados por esta última.

*   **Formatos de resultado:** Pueden devolver un valor único (**escalar**), una sola fila con múltiples columnas, o una tabla completa con múltiples filas y columnas.

*   **Independencia:** Hay de dos tipos.
    *   **No correlacionadas:** Son independientes y pueden ejecutarse por sí solas sin referenciar nada de la consulta externa.
    *   **Correlacionadas:** Dependen de la consulta externa, ya que hacen referencia a columnas de esta. Se ejecutan una vez por cada fila candidata de la consulta principal.

---

### Filtrar datos con subconsulta

Para filtrar datos utilizando una subconsulta en la cláusula **`WHERE`**, se inserta una instrucción `SELECT` completa entre paréntesis dentro de la condición de filtrado de la consulta principal. La subconsulta actúa como un generador de valores dinámicos que la consulta externa utiliza para comparar y decidir qué filas incluir en el resultado final.

Existen varias formas de aplicar este filtro dependiendo del tipo de resultado que devuelva la subconsulta:

#### Subconsultas Escalares (Valor único)
Se utilizan cuando la subconsulta devuelve una **sola columna y una sola fila**. En este caso, puedes usar operadores de comparación estándar como `=`, `<>`, `>`, `<`, `>=`, o `<=`.

*   **Ejemplo:** Encontrar al cliente con el ID más alto.
    ```sql
    SELECT customer_id, first_name, last_name
    FROM customer
    WHERE customer_id = (SELECT MAX(customer_id) FROM customer);
    ```
    *Aquí, la subconsulta obtiene el valor máximo (por ejemplo, 599) y la consulta principal filtra por ese número específico.*

#### Subconsultas de Lista (Múltiples filas, una columna)
Si la subconsulta devuelve una lista de valores, se deben utilizar operadores especiales diseñados para conjuntos:

*   **`IN` / `NOT IN`:** Comprueba si un valor se encuentra (o no) dentro del conjunto devuelto.
    ```sql
    SELECT city
    FROM city
    WHERE country_id IN (SELECT country_id FROM country WHERE country IN ('Canada', 'Mexico'));
    ```
*   **`ANY` / `SOME`:** Devuelve verdadero si la comparación es cierta para **al menos uno** de los valores de la lista.
*   **`ALL`:** Devuelve verdadero solo si la comparación es cierta para **todos** los valores devueltos.

#### Subconsultas de Existencia (`EXISTS`)
Este operador no busca un valor específico, sino que simplemente verifica si la subconsulta devuelve **al menos una fila**. Es muy útil para validar relaciones entre tablas sin importar la cantidad.

*   **Ejemplo:** Listar clientes que realizaron al menos un alquiler antes de una fecha específica.
    ```sql
    SELECT first_name, last_name
    FROM customer c
    WHERE EXISTS (SELECT 1 FROM rental r 
                  WHERE r.customer_id = c.customer_id 
                  AND date(r.rental_date) < '2005-05-25');
    ```
    *En este caso, se usa una **subconsulta correlacionada**, donde el filtro interno hace referencia a una columna de la consulta externa (`c.customer_id`).*

#### Subconsultas de Múltiples Columnas
Aunque menos comunes, puedes comparar varias columnas a la vez envolviéndolas en paréntesis (si el motor de base de datos lo soporta):
```sql
SELECT actor_id, film_id
FROM film_actor
WHERE (actor_id, film_id) IN (SELECT a.actor_id, f.film_id FROM actor a CROSS JOIN film f WHERE a.last_name = 'MONROE');
```

#### Reglas importantes a recordar:
*   **Paréntesis:** La subconsulta siempre debe ir encerrada entre paréntesis.

*   **Orden de ejecución:** Generalmente, la subconsulta se ejecuta primero para pasar sus resultados a la consulta externa.

*   **Manejo de NULLs:** Si usas `NOT IN` y la subconsulta devuelve un valor `NULL`, la consulta principal podría no devolver ningún resultado, por lo que se recomienda usar `NOT EXISTS` en esos casos.

*   **Rendimiento:** Las subconsultas correlacionadas pueden ser más lentas porque se ejecutan una vez por cada fila procesada en la consulta principal.


### Operadores en subconsultas de lista

Para filtrar datos utilizando subconsultas que devuelven una lista de valores (múltiples filas y una sola columna), se utilizan operadores diseñados específicamente para manejar conjuntos de resultados.

A continuación se detallan los operadores principales:

#### Operadores de Pertenencia (`IN` y `NOT IN`)
Son los más utilizados para comparar un valor individual contra el conjunto devuelto por la subconsulta.
*   **`IN`**: Evalúa como verdadero si el valor de la columna coincide con **al menos uno** de los valores de la lista. Es semánticamente equivalente a usar una serie de condiciones unidas por el operador `OR`.

*   **`NOT IN`**: Evalúa como verdadero si el valor **no se encuentra** en la lista generada por la subconsulta.

#### Operadores Cuantificados (`ALL`, `ANY`, `SOME`)
Estos operadores se utilizan junto con los operadores de comparación estándar (`=`, `<>`, `<`, `>`, `<=`, `>=`) para evaluar la relación entre un valor y la lista completa.

*   **`ALL`**: La condición se cumple solo si la comparación es verdadera para **todos** los valores de la lista. Por ejemplo, `> ALL` significa que el valor debe ser mayor que el valor máximo devuelto por la subconsulta.

*   **`ANY` / `SOME`**: La condición es verdadera si la comparación se cumple para **al menos uno** de los valores de la lista. Los términos `ANY` y `SOME` son sinónimos y funcionan de la misma manera.

#### Equivalencias Clave
Las fuentes señalan que existen redundancias funcionales entre estos operadores:
*   Usar **`IN`** es equivalente a usar **`= ANY`**.
*   Usar **`NOT IN`** es equivalente a usar **`<> ALL`**.

#### Consideraciones sobre valores `NULL`
Es fundamental tener precaución al usar `NOT IN` o `<> ALL` si la subconsulta puede devolver un valor `NULL`. Si existe un `NULL` en la lista, la comparación puede resultar en un valor "desconocido" (unknown), lo que a menudo provoca que la consulta principal no devuelva ningún resultado. En estos casos, se suele recomendar el uso del operador `EXISTS` o `NOT EXISTS` como alternativa más robusta.

### Ejemplos

#### 1. Subconsulta en la cláusula `WHERE` (Filtrado)
Es el uso más común, donde la subconsulta proporciona un valor o lista para comparar.

*   **Ejemplo escalar (valor único):** Para encontrar al cliente con el ID más alto:
    ```sql showLineNumbers
    SELECT customer_id, first_name, last_name
    FROM customer
    WHERE customer_id = (SELECT MAX(customer_id) FROM customer);
    ```
*   **Ejemplo con `IN` (conjunto de valores):** Para listar ciudades que pertenecen a Canadá o México:
    ```sql showLineNumbers
    SELECT city_id, city
    FROM city
    WHERE country_id IN (SELECT country_id FROM country WHERE country IN ('Canada', 'Mexico'));
    ```

#### 2. Subconsulta en la cláusula `SELECT` (Generador de expresiones)
Se usa para crear una columna calculada basándose en datos de otra tabla sin necesidad de un `JOIN` explícito en la consulta principal.

*   **Ejemplo:** Listar clientes y el número total de pedidos que cada uno ha realizado:
    ```sql showLineNumbers
    SELECT c.CustFirstName, c.CustLastName,
    (SELECT COUNT(*) FROM Orders o WHERE o.CustomerID = c.CustomerID) AS CountOfOrders
    FROM Customers c;
    ```

#### 3. Subconsulta en la cláusula `FROM` (Tablas derivadas)
La subconsulta genera un conjunto de resultados temporal que la consulta externa trata como si fuera una tabla física.

*   **Ejemplo:** Unir una tabla de clientes con una subconsulta que resume sus pagos:
    ```sql showLineNumbers
    SELECT c.first_name, c.last_name, pymnt.num_rentals
    FROM customer c
    INNER JOIN (
        SELECT customer_id, count(*) AS num_rentals
        FROM payment
        GROUP BY customer_id
    ) AS pymnt ON c.customer_id = pymnt.customer_id;
    ```

#### 4. Subconsulta correlacionada con `EXISTS`
Se utiliza para verificar si existe una relación, sin importar la cantidad.

*   **Ejemplo:** Encontrar actores que nunca han aparecido en una película clasificada como "R":
    ```sql showLineNumbers
    SELECT a.first_name, a.last_name
    FROM actor a
    WHERE NOT EXISTS (
        SELECT 1 FROM film_actor fa
        INNER JOIN film f ON f.film_id = fa.film_id
        WHERE fa.actor_id = a.actor_id AND f.rating = 'R'
    );
    ```

#### 5. Subconsulta en instrucciones de modificación (`UPDATE`, `DELETE`)
Permiten seleccionar de forma precisa qué filas modificar o eliminar basándose en criterios complejos.

*   **Ejemplo de actualización:** Actualizar la fecha de última actividad de un cliente basándose en su alquiler más reciente:
    ```sql showLineNumbers
    UPDATE customer c
    SET c.last_update = (SELECT MAX(r.rental_date) FROM rental r WHERE r.customer_id = c.customer_id);
    ```

#### Otros ejemplos

Distribución de ordenes por cliente.

```sql showLineNumbers
SELECT orders, count(*) as num_customers 
FROM
(
   SELECT customer_id, count(order_id) as orders 
   FROM orders
   GROUP BY 1
) a
GROUP BY 1
```

Conocer el producto mas caro.

```sql showLineNumbers
SELECT nombre, codigo, precio
FROM productos
WHERE precio=(SELET max(precio) FROM productos)
```

Las subconsultas que retornan un solo valor escalar se utiliza con un operador de comparación o en lugar de una expresión

```sql showLineNumbers
SELECT CAMPOS
FROM TABLA
WHERE CAMPO OPERADOR (SUBCONSULTA)

SELECT CAMPO OPERADOR (SUBCONSULTA)
FROM TABLA
```

Conocer el valor de un artículo determinado ('pesa') y la diferencia con el artículo mas costoso.

```sql showLineNumbers
SELECT articulo, precio,
precio - (SELECT max(precio) FROM libros) as diferencia
FROM productos
WHERE articulo = 'pesa'
```

Actualizamos el precio del producto con el valor mayor:.

```sql showLineNumbers
UPDATE productos SET precio=45
WHERE valor = (SELECT max(valor) FROM productos)
```
