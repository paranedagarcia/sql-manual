Una **tabla pivot** (también conocida como tabla dinámica, tabulación cruzada o *crosstab*) es una herramienta de resumen que permite transformar datos organizados en filas en un formato de columnas. Su objetivo es presentar la información en una **matriz o cuadrícula** donde las filas representan una variable, las columnas otra, y las celdas en su intersección contienen un valor agregado (como una suma, un promedio o un recuento).

Una tabla de pivoteo es una manera de resumir datos en forma de filas y columnas. Dado por los atributos de una columna donde la intersección de fila y columna se utiliza un valor de tipo estadístico como suma , cuenta o promedio. Cuadro resumen de las ventas de cada producto por fecha.

Algunos métodos y ejemplos para generar tablas pivot:

### Uso de expresiones `CASE` y funciones de agregado
Esta es la técnica más común y portable entre diferentes motores de SQL. Consiste en agrupar por la variable de fila y usar una función de agregado (como `SUM` o `COUNT`) combinada con un `CASE` para crear las columnas.

**Ejemplo:** Supongamos una tabla de pedidos (`orders`) con columnas `order_date`, `product` y `order_amount`. Para ver las ventas totales por fecha divididas en columnas por tipo de producto (camisa y zapatos), se usaría:

```sql showLineNumbers
SELECT order_date
    ,SUM(CASE WHEN product = 'shirt' THEN order_amount ELSE 0 END) AS shirts
    ,SUM(CASE WHEN product = 'shoes' THEN order_amount ELSE 0 END) AS shoes
FROM orders
GROUP BY 1;
```


### Función `crosstab()` en PostgreSQL
PostgreSQL ofrece un módulo llamado `tablefunc` que debe instalarse previamente con el comando `CREATE EXTENSION tablefunc;`. La función `crosstab()` toma una consulta SQL como entrada y genera la matriz automáticamente.

**Ejemplo práctico (Encuesta de helados):** Si tienes resultados de una encuesta por oficina y sabor favorito, puedes tabularlos así:

```sql showLineNumbers
SELECT *
FROM crosstab('SELECT office, flavor, count(*) FROM ice_cream_survey GROUP BY office, flavor ORDER BY office',
              'SELECT flavor FROM ice_cream_survey GROUP BY flavor ORDER BY flavor')
AS (office varchar(20), chocolate bigint, strawberry bigint, vanilla bigint);
```
Este comando genera una tabla donde cada fila es una oficina y cada columna es un sabor con el conteo de votos respectivo.

### 3. Cláusula `PIVOT` (SQL Server y Oracle)
Motores como SQL Server y Oracle incluyen una sintaxis específica llamada `PIVOT` para simplificar estas consultas.

**Ejemplo de sintaxis:**
```sql showLineNumbers
SELECT *
FROM orders
PIVOT(SUM(order_amount) FOR product IN ('shirt', 'shoes')) AS PivotTable;
```


### Uso de "Tablas Driver" (Tablas de ayuda)
En algunos casos, se utilizan "tablas driver" o de ayuda (como una tabla que solo contenga los meses del año) para forzar que el resultado muestre todas las categorías deseadas, incluso si no hay datos para ellas en la tabla principal. Se realiza un `CROSS JOIN` entre la tabla driver y los datos para asegurar que cada fila de datos se combine con cada valor de la columna pivotante.

### Resumen
Las tablas pivot son especialmente útiles para:
*   **Audiencias de negocios:** Resumen datos complejos en formatos compactos similares a hojas de cálculo de Excel.
*   **Análisis de series temporales:** Comparar métricas mes a mes o año a año en columnas adyacentes.
*   **Normalización de datos dispersos:** Ensanchar tablas que originalmente son largas y estrechas para facilitar su visualización.

**Nota de rendimiento:** Generar tablas pivot, especialmente con la función `crosstab()`, puede ser intensivo para el procesador en conjuntos de datos que contienen millones o miles de millones de filas.

### Ejemplos

```sql showLineNumbers
SELECT fecha, 
sum(case WHEN product = 'polera' THEN monto ELSE 0 END) as poleras,
sum(case WHEN product = 'camisa' THEN monto ELSE 0 END) as camisas,
sum(case WHEN product = 'zapato' THEN monto ELSE 0 END) as zapatos
FROM ordenes
GROUP BY 1
```

Promedio de valor de propiedades por comuna y numero de dormitorios.

```sql showLineNumbers
SELECT comuna,
   avg(valor) FILTER (WHERE dormitorio = 1) AS "1",
   avg(valor) FILTER (WHERE dormitorio = 2) AS "2",
   avg(valor) FILTER (WHERE dormitorio = 3) AS "3",
   avg(valor) FILTER (WHERE dormitorio = 4) AS "4"
FROM propiedades
GROUP BY comuna;
```

```sql showLineNumbers
SELECT * FROM   
(
    SELECT 
        category_name, 
        product_id
    FROM 
        production.products p
        INNER JOIN production.categories c 
            ON c.category_id = p.category_id
) t 
PIVOT(
    COUNT(product_id) 
    FOR category_name IN (
        [Children], 
        [Comfort], 
        [Cruisers], 
        [Cyclocross], 
        [Electric], 
        [Mountain], 
        [Road])
) AS pivot_table;
```

Agregando el año del modelo.

```sql showLineNumbers
SELECT * FROM   
(
    SELECT 
        category_name, 
        product_id,
        model_year
    FROM 
        production.products p
        INNER JOIN production.categories c 
            ON c.category_id = p.category_id
) t 
PIVOT(
    COUNT(product_id) 
    FOR category_name IN (
        [Children], 
        [Comfort], 
        [Cruisers], 
        [Cyclocross], 
        [Electric], 
        [Mountain], 
        [Road])
) AS pivot_table;
```

En PostgreSQL se requiere habilitar la función tablefunc (`CREATE` `extension tablefunc;`) para utilizar crosstab.

Seleccionar el puntaje obtenido de cada alumno en cada examen.

```sql showLineNumbers
SELECT * 
FROM crosstab(
'SELECT nombre, examen, puntaje 
 FROM examenes ORDER BY 1,2')
AS notas (nombre varchar(15),examen1 int, examen2 int, 
examen3 int, examen4 int);
```
