En SQL, la operación **JOIN** permite combinar datos de dos o más tablas basándose en una relación entre columnas comunes. La diferencia principal entre los tipos de JOIN radica en cómo manejan las filas que **no tienen una coincidencia** exacta en la otra tabla.

A continuación se detallan las diferencias entre los tipos de JOIN:

### 1. INNER JOIN (Unión interna)
Es el tipo de JOIN más común y el más restrictivo.
*   **Funcionamiento:** Devuelve únicamente las filas donde existe una coincidencia en ambas tablas según el predicado de unión.
*   **Resultado:** Si una fila de la "Tabla A" no tiene una pareja correspondiente en la "Tabla B", esa fila simplemente se descarta del resultado final.

### 2. OUTER JOIN (Unión externa)
A diferencia del anterior, este tipo de JOIN preserva las filas que no tienen coincidencia, rellenando las columnas de la tabla faltante con valores **NULL**. Se dividen en tres:
*   **LEFT (OUTER) JOIN:** Devuelve todas las filas de la tabla de la izquierda y las filas coincidentes de la derecha. Si no hay coincidencia, las columnas de la tabla derecha aparecen como NULL.
*   **RIGHT (OUTER) JOIN:** Es el inverso del anterior; devuelve todas las filas de la tabla de la derecha y las coincidentes de la izquierda. Las fuentes indican que es menos común que el LEFT JOIN y que no todos los servidores lo soportan igual.
*   **FULL (OUTER) JOIN:** Combina los efectos de LEFT y RIGHT JOIN. Devuelve todas las filas de ambas tablas, uniendo las que coinciden y manteniendo las que no (con valores NULL en los huecos). **Nota:** MySQL no soporta el FULL JOIN directamente.

### 3. CROSS JOIN (Unión cruzada)
También conocido como producto cartesiano.
*   **Funcionamiento:** Empareja cada fila de la primera tabla con todas y cada una de las filas de la segunda.
*   **Resultado:** Si la Tabla A tiene 10 filas y la Tabla B tiene 100, el resultado tendrá 1,000 filas (10x100). Generalmente no requiere una condición de unión (cláusula `ON`).

### 4. SELF JOIN (Autounión)
No es un comando distinto, sino una técnica donde una tabla se une **consigo misma**.
*   **Uso:** Es útil para consultar datos jerárquicos dentro de una misma tabla, como una tabla de empleados donde una columna indica quién es el jefe (que también es un empleado en la misma tabla).

Un **SELF JOIN** es una técnica en la que una tabla se une **consigo misma**. No es un comando distinto, sino que se utiliza cualquier tipo de JOIN (generalmente `INNER` o `LEFT JOIN`) tratando a la misma tabla como si fueran dos entidades diferentes mediante el uso de **alias**.

A continuación, se presentan los dos ejemplos prácticos más comunes:

### 1. Relación de Jerarquía (Empleados y Jefes)
Este es el uso más habitual. Imagine una tabla llamada `Employees` donde cada fila contiene un `EmployeeID` (el ID del empleado) y un `ManagerID` (el ID de su jefe, que también es un empleado en la misma tabla).

Para obtener una lista de empleados junto al nombre de su respectivo jefe, se realiza la siguiente consulta:

```sql
SELECT 
    e.FirstName + ' ' + e.LastName AS NombreEmpleado,
    m.FirstName + ' ' + m.LastName AS NombreJefe
FROM Employees AS e
LEFT OUTER JOIN Employees m 
ON e.ManagerID = m.EmployeeID
ORDER BY NombreJefe;
```
**Puntos clave de este ejemplo:**
*   **Uso de Alias:** Se asignan alias (`e` para empleados y `m` para jefes) para que el servidor pueda distinguir a qué "instancia" de la tabla se refiere cada columna.
*   **Condición de Unión:** La unión se hace donde el `ManagerID` de la instancia de empleados coincida con el `EmployeeID` de la instancia de jefes.
*   **LEFT JOIN:** Se prefiere usar `LEFT OUTER JOIN` en lugar de `INNER JOIN` para no excluir al empleado de mayor rango (como el CEO), quien no tiene un jefe asignado (`NULL`).

### 2. Relación de Referencia (Películas y Precuelas)
Otro ejemplo práctico ocurre cuando una tabla de películas tiene una columna que apunta a otra película de la misma tabla, como una precuela. Supongamos que la tabla `film` tiene la columna `prequel_film_id`.

La consulta para listar cada película con su precuela sería:

```sql
SELECT 
    f.title AS Pelicula, 
    f_prnt.title AS Precuela
FROM film f
INNER JOIN film f_prnt
ON f_prnt.film_id = f.prequel_film_id
WHERE f.prequel_film_id IS NOT NULL;
```
En este caso, si la película "Fiddler Lost II" tiene un `prequel_film_id` que apunta al ID de "Fiddler Lost", el resultado mostrará ambos títulos en la misma fila.

### Resumen
El **SELF JOIN** es indispensable para:
*   Consultar **datos jerárquicos** (organigramas).
*   Comparar filas dentro de la **misma tabla** (por ejemplo, buscar empleados que vivan en la misma ciudad).
*   Analizar **series de datos** o relaciones de precedencia (precuelas, requisitos previos de cursos, etc.).


### 5. NATURAL JOIN (Unión natural)
Es una forma "automática" de realizar un JOIN.
*   **Funcionamiento:** El servidor de la base de datos inspecciona las tablas y une aquellas columnas que tengan exactamente el mismo nombre.
*   **Riesgo:** Puede ser peligroso si las tablas comparten nombres de columnas que no son llaves relacionadas (por ejemplo, si ambas tienen una columna llamada `fecha_actualizacion`), lo que podría devolver un conjunto de resultados vacío o incorrecto.

### Resumen de diferencias clave
| Tipo de JOIN | Filas devueltas | Manejo de no coincidencias |
| :--- | :--- | :--- |
| **INNER** | Solo coincidencias en ambas tablas. | Se eliminan. |
| **LEFT** | Todas las de la izquierda + coincidencias derecha. | NULL en lado derecho. |
| **RIGHT** | Todas las de la derecha + coincidencias izquierda. | NULL en lado izquierdo. |
| **FULL** | Todas las filas de ambas tablas. | NULL donde falte el dato. |
| **CROSS** | Todas las combinaciones posibles (producto). | No aplica (une todo). |
